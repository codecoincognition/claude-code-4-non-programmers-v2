## Decisions made (decision + why)

1. **LinkedIn OAuth uses a User-Agent header fix.**
   *Why:* LinkedIn's OAuth endpoint has a redirect loop that resets the cookie
   if the request comes from a non-browser User-Agent. We worked this out by
   reading the error trace in dashboard-deploy.log. The fix is to send a
   browser-shaped User-Agent header on the auth request. Without the *why*,
   the bare summary "fixed LinkedIn OAuth" is not reconstructable.

2. **The Stripe refresher needs the same shape with one variation.**
   *Why:* Stripe's endpoint also rejects non-browser agents, but needs a
   different UA string than LinkedIn's. Do not copy LinkedIn's UA verbatim.

3. **Cache reads come from the watchdog mesh, not direct scrape.**
   *Why:* The Ch 21 dashboard scraped directly; we moved to reading a cache the
   watchdog mesh populates, to avoid hammering the source on every refresh.
