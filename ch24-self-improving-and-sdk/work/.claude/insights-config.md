# /insights-to-skills configuration

## Gate at commit (load-bearing)

Before authoring any skill, ask:
  "Are you going to keep doing this for the next quarter,
   or is this a habit you're about to change?"

Default answer if no response in 60 seconds: NO. Don't author.

Rationale: better to under-author than over-author. Skills for
abandoned habits accumulate as drift; missing skills cost only
re-prompting until the next /insights run.

## Other policies

- Minimum repetition for skill candidacy: 5 occurrences in 30 days
- Maximum skills authored per /insights-to-skills run: 1
  (forces deliberation; no batch-approving 8 skills at once)
- Skills authored by /insights-to-skills tagged with provenance
  ("# generated by /insights-to-skills on YYYY-MM-DD")
  so audit at quarterly governance hour (Ch 26) can find them.
