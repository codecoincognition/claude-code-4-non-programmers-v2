# Fixtures — input shapes for Chapter 24

The artifacts in this chapter read from your own accounts and history.
We don't ship private data. This file documents the input shapes plus a
tiny synthetic sample where it helps.

## 1. Session history (read by `/insights`)

`/insights --last 30d` reads your Claude Code session history — the
prompts you typed, the commands you ran, the agents that fired. This
lives in your local Claude Code state, not in a file you commit. There
is nothing to clone here. When you run `/insights` against your own
30/90-day window it produces a report like the one in the chapter:
patterns ranked by impact, each routed to the right surface (skill,
hook, slash command, hint, or "no action needed").

## 2. Iris's Notion output (read by the meta-judge)

The `meta-judge` agent reads Iris's last 24 hours from two Notion
surfaces:

- **Tasks DB rows** — one row per routed thread. Relevant fields:
  - `Title` (thread subject)
  - `Routing` (one of: `action`, `FYI`, `commit`)
  - `Drafted reply` (text, if Iris drafted one)
  - `Manual tag` (set by Devon; meta-judge checks it was respected)
  - `Source thread URL` / Notion page URL (for the inline link in the ping)
- **Project Notes pages** — longer-form notes Iris wrote for `commit`-class threads.

A synthetic sample of three Tasks DB rows is in
`iris-tasks-sample.json` so you can see the shape the meta-judge grades
against. Replace with your own Notion data via the Notion MCP.

## 3. Slack (written by the meta-judge and the SDK build)

The meta-judge posts disagreement pings to `#devon-watchdog`. The SDK
build posts deploy URLs to `#devon-deploys`. Both go through the Slack
MCP / a Slack bot token. No fixture — these are write surfaces.

## 4. The declined skill (`friday-week-summary.md`)

The chapter has Devon decline this skill at the gate ("changing in two
weeks; don't generate"). It is therefore **not created** — that's the
point of the When-it-goes-wrong beat. We don't ship it as a real skill.
Had he approved it, the draft would have looked like:

```
---
description: Summarize my week and file the summary under journal/.
trigger: Friday 5 PM
tools: Read, Write
# generated by /insights-to-skills on 2026-05-30
---

# Friday week summary

Summarize my week from this week's sessions and notes, then file the
summary at journal/YYYY-WW.md.
```

It lives here only as documentation of what the gate prevented.
